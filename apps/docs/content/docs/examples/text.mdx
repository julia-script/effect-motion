`Shapes.Text` mirrors SVG `<text>`. Its required `text` field is a plain
string. Everything else has visible defaults: 16px, the generic `sans-serif`
family, and black fill.

## Plain text

```ts
const label = yield* Scene.instantiate(Shapes.Text, {
	text: "Hello, world",
	x: 40,
	y: 80,
});
```

The engine doesn't measure text (no wrapping, no fit-to-box), so alignment
is delegated to SVG: `textAnchor: "middle"` + `baseline: "middle"` centers
the whole Text on its `x`/`y`. Entity-level properties such as `fill`,
`opacity`, and `fontSize` apply to the whole Text; `fontSize` is tweenable.

## Composing text

`Text` is a leaf — it holds a string, not a formatting tree. Styled runs and
multi-line captions are built by composing multiple `Text` instances under a
`Group`, using the polymorphic `children` list. A bare string in `children`
becomes a `Text`:

```ts
yield* Scene.instantiate(Shapes.Group, {
	x: 40,
	y: 80,
	children: [
		"effect-",
		Scene.instantiate(Shapes.Text, { text: "motion", x: 60, fill: "#7f5af0" }),
	],
});
```

Inline formatting (bold/italic mid-line) and markdown authoring are handled in
userland — build the `Group`/`Text` instances yourself (a builder you can
`memoize` outside the scene, reusing any markdown library). The engine renders
the tree; it does not parse content.

## Hiding a node

Every instance carries a builtin `$visible` property (default `true`), set at
instantiation. A hidden instance is skipped by renderers:

```ts
yield* Scene.instantiate(Shapes.Text, { text: "hidden for now", $visible: false });
```

## Custom fonts

Text only ever *names* a family — making that name resolve is the job of
whatever displays the frames. A scene declares the fonts it depends on with
the `Fonts` annotation, one entry per face (declare bold or italic faces as
separate entries with `weight`/`style`):

<Example name="custom-fonts" />

The runtime never reads the annotation — fonts cannot affect frame data —
but each consumer honors it its own way:

- **The player** loads every entry with a `src.url` through the browser's
  `FontFace` API while it buffers the first frames, and reports `ready` only
  once the loads settle, so the first visible frame is drawn with the
  declared fonts. A font that fails to load logs a warning and playback
  proceeds with the browser's normal fallback.
- **Export** maps every entry with a `src.path` to resvg's font files via
  `Fonts.resvgOptions` from `@effect-motion/export`:

```ts
import { Fonts, Resvg } from "@effect-motion/export";

yield* Resvg.rasterize(svg, Fonts.resvgOptions(scene));
```

System fonts stay available by default; pass
`font: { loadSystemFonts: false }` alongside the mapped options when the
declared files should be the only ones resvg sees. Pointing `url` and `path`
at the same font is what makes browser playback and exported frames render
with matching families.
