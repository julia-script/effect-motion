---
title: Exporting to video
description: Turn a scene into an MP4 file with @effect-motion/export and ffmpeg
---

A scene is a stream of frames. `@effect-motion/export` turns that stream into
a shareable artifact: each frame is rendered to an SVG string, rasterized to
PNG with [resvg](https://github.com/linebender/resvg), and piped into
[ffmpeg](https://ffmpeg.org) to produce an MP4 — no intermediate files ever
touch disk.

## One call: scene to file

`Video.render` composes the whole pipeline. It reads the framerate and
dimensions from the scene's own frame metadata, so you never repeat them.

```ts
import { Effect } from "effect"
import { NodeServices } from "@effect/platform-node"
import { Video } from "@effect-motion/export"
import { myScene } from "./scene"

await Effect.runPromise(
  Video.render(myScene, "out.mp4", {
    settings: { frameRate: 60 },
  }).pipe(Effect.provide(NodeServices.layer)),
)
```

`NodeServices.layer` (from `@effect/platform-node`) provides the process
spawner ffmpeg runs through. The output is broadly playable by default:
H.264, `yuv420p`, faststart MP4.

### Options

- **`frames`** — cap the number of frames encoded. An **infinite scene** must
  set this: without it, the encode runs as long as the scene does, forever.
- **`settings`** — the scene's `frameRate` and
  `maxFrames` for this export. Set `maxFrames: Infinity` for an intentionally
  infinite scene (paired with `frames`).
- **`binary`** — override the ffmpeg binary. Defaults to the **bundled**
  `ffmpeg-static` build; pass `"ffmpeg"` (or a path) to use a system/custom
  ffmpeg instead.
- **`extraArgs`** — extra ffmpeg arguments appended before the output path,
  e.g. `["-crf", "18"]` for higher quality, or a `-vf` scale filter.

### Even dimensions

`yuv420p` requires even width and height. `Video.render` checks the first
frame and fails with an `EncodeError` *before* spawning ffmpeg if either is
odd — no silent resizing. Use even dimensions, or pass a scale filter through
`extraArgs`.

## Fonts

Fonts are typed scene dependencies (see [Fonts](/docs/going-further/fonts)):
provide the same loader layers to the export path's render that the player
gets via `renderLayers`, and exported frames use the same faces the player
shows. In Node, load effects can read font bytes straight from disk — the
offline-rendering path needs no URLs.

## ffmpeg is bundled

Video encoding uses the bundled `ffmpeg-static` binary by default — a full build
that includes **libx264**, so H.264 output works out of the box with no system
ffmpeg required (installing `@effect-motion/export` downloads that binary for
your platform). To use a system or custom ffmpeg instead, pass `binary`
(`"ffmpeg"`, or a path). If the chosen binary can't run, `Video.render` fails
with an `EncodeError`.

> The bundled ffmpeg build (and libx264) is GPL-licensed — a standalone
> executable invoked over a process boundary, not linked into your code. Pass
> your own `binary` if you'd rather not ship it.

## Lower-level building blocks

`Video.render` is the composed path. Its pieces are also exported for custom
pipelines:

- `Resvg.rasterize(svg, options?)` — SVG string → PNG bytes.
- `Ffmpeg.encode(pngStream, outPath, options)` — a `Stream` of PNG frames →
  a video file, requiring a `ChildProcessSpawner`.
