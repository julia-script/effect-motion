# Type Alias: Target\<Tag\>

```ts
type Target<Tag> = 
  | Partial<InterpolableOrInterpolator<EntityByTag<Tag>>>
  | ((data) => Partial<InterpolableOrInterpolator<EntityByTag<Tag>>>);
```

Defined in: [packages/motion/src/Motion.ts:157](https://github.com/julia-script/effect-motion/blob/main/packages/motion/src/Motion.ts#L157)

What [tween](../variables/tween.mdx) / [tweenTo](../variables/tweenTo.mdx) accept as an endpoint: the fields to
animate and their values.

## Type Parameters

| Type Parameter | Description |
| ------ | ------ |
| `Tag` *extends* [`EntityTag`](../../Entity/type-aliases/EntityTag.mdx) | The entity tag, which determines the available fields. |

## Remarks

Two forms. A plain object states targets literally. A function receives
the entity's data at animation start and computes them, which is how you
express a target relative to where the entity already is (`data.radius *
2`) without reading it out in a separate step.

Only numeric and Color fields of the entity are addressable — those are
the ones that can be interpolated.
