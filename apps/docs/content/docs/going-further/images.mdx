title: Images
description: Typed image dependencies, and placing them with Shapes.Image

Images are **typed scene dependencies**, exactly like fonts: declare a
constant, yield it for the value the entity stores, and the scene's type
carries an `ImageLoader` requirement that rendering will demand:

```ts
import { Image, Resource, Scene, Shapes } from "effect-motion";

const Rocket = Image.Image("rocket");

const scene = Scene.make(function* () {
	const rocket = yield* Rocket;
	yield* Scene.instantiate(Shapes.Image, { image: rocket, width: 72, height: 72 });
});
// typeof scene: Scene<never, ImageLoader<"rocket"> | Runner>

const layers = Image.layer(Rocket, Resource.fetchBytes("https://…/rocket.png"));
```

The layer's load effect runs once at mount; the render session decodes the
bytes once, and every frame reuses the decoded picture. Supported formats
are what the engine decodes: PNG, JPEG, WebP, and SVG. An image referenced
at render with no loader in context is a defect naming the id; a load
failure surfaces where the runtime is built (the player's error state) —
nothing fails silently at frame time. Frames stay deterministic regardless:
decoded pixels never affect frame data.

## Sizing

`width` and `height` are optional:

- **Both set** — the image is drawn at exactly that size. They're plain
  numeric fields, so scaling an image over time is just `Motion.tweenTo` with
  `width`/`height` targets.
- **Both absent** — the image draws at its source's natural size.
- **Only one set** — it's ignored (natural size is used). Aspect-preserving
  single-dimension scaling would need the natural size, which frame data
  never sees.

`x`/`y` place the image's top-left corner, like `Rect`. Images are
billboards: they follow the camera's pan, zoom, and depth scaling, but unlike
`Rect` they can't tilt in 3D.

## Choosing sources

A load effect is any `Effect<Uint8Array>`: `Resource.fetchBytes(url)` in the
browser (CORS-open for cross-origin URLs; pin CDN versions), a filesystem
read in Node export paths. A broken source fails the layer build loudly —
you find out at mount, not by an image quietly not drawing.
