title: Fonts

Fonts are **typed scene dependencies**. Declaring one mints a constant you
yield inside the scene; the yield returns the value entity props store, and
adds a `FontLoader` requirement to the scene's type:

```ts
import { Font, Scene, Shapes } from "effect-motion";

const Pacifico = Font.Font("Pacifico");

const scene = Scene.make(function* () {
	const pacifico = yield* Pacifico;
	yield* Scene.instantiate(Shapes.Text, {
		text: "hello",
		fontFamily: pacifico,
	});
});
// typeof scene: Scene<never, FontLoader<"Pacifico"> | Runner>
```

Running the scene never needs the font — frames are pure of resource bytes
(the engine cannot measure text). Only **rendering** does: the frames carry
the requirement as `Frame<FontLoader<"Pacifico">>`, and `Renderer.render`
(and therefore the player) will not compile until a covering layer is
provided:

```ts
import { Font, Resource } from "effect-motion";

const layers = Font.layer(
	Pacifico,
	Resource.fetchBytes("https://…/Pacifico_400Regular.ttf"),
);

<Player scene={scene} renderLayers={layers} />;
```

The load effect runs **once, at mount** — before anything renders — so the
first visible frame is already in the declared font. Compose retries or
timeouts directly on the load effect. Every provided font loads whether or
not the scene ends up using it. A font referenced at render with no loader
in context is a defect naming the font — there is no silent glyph fallback.

The renderer rasterizes TrueType/OpenType bytes (`.ttf` / `.otf`, sniffed
from the magic bytes) — point load effects at a `.ttf` asset, not woff2. In
Node (export paths), a load effect can read from disk:
`Effect.promise(() => readFile(path).then((b) => new Uint8Array(b)))`.

## The default font

`Text.fontFamily` defaults to the built-in `Font.default` under the
**reserved id `"sans-serif"`**. It never appears in a scene's requirements
(no yield happens for a schema default), and the render path auto-provides
its loader — bare `Text` and string children work with zero ceremony.

Providing your own loader under the `"sans-serif"` id **overrides** the
built-in bytes — that collision is the override mechanism, not an error:

```ts
// every default-font text now renders in your bytes
Font.layer(Font.Font("sans-serif"), Resource.fetchBytes(myFontUrl));
```
